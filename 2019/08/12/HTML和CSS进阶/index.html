background

[background-attachment]
图片是否固定

  • fixed——页面滚动，背景不动
  • scroll——背景随页面滚动

[background-position]

  • 背景图片小于容器时，背景图片的起始位置
  • 五个可选值和数值
  • top、bottom、left、right、center

[background-size]

  • 设置背景的大小
  • 数值或百分比
  • cover——扩展至整个容器大小，可能超出容器范围
  • contain——保持纵横比并且缩放至宽度或高度适应容器

[background-origin]

  • background-position的参照
  • 没什么用，调整background-position数值就好了

[background-clip]

  • 背景是否填充到边框盒或内边距盒或内容盒
  • 没什么用

注意点
①background-position和background-size用斜杠分隔：
两个属性的值非常的类似，比如都可以使用如下形式：

两个属性值都规定的话，需要用斜杠分隔。
如果规定一个，那么仅对background-position属性有效，background-size采用默认值。

②同时规定背景色与背景图片，图片会覆盖在颜色之上。
③对于各个子属性的顺序官方并没有强制要求，background-position和background-size两个属性不就固定顺序了吗，既然它们之间需要用斜线分隔了，可以看做一个整体处理。
④为什么设置height：100%无效

https://blog.csdn.net/yanchenxi313761/article/details/79437541

浏览器是不会自动计算元素的高度的，设置height：100%必须要求父元素有固定高度，如果你的父元素也没有固定高度，那么一直上溯就需要设置html标签的height是100%了。

所以解决办法就是父元素和自身都设置height：100%
